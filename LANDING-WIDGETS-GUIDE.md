## üéØ High-Priority Widgets

### 1. Comparison Table Widget
**Widget Name:** `comparison-table`  
**Icon:** `table`  
**Best For:** Cost comparisons, feature comparisons, plan comparisons

**Key Features:**
- Multiple columns (typically 2)
- Highlighted "winner" column with custom color
- Row-by-row item comparison
- Totals/summary row
- Fully responsive

**Perfect For:**
- Traditional vs Platform cost comparison
- Service tier comparisons
- Feature availability across plans

---

### 2. Process Steps Widget
**Widget Name:** `process-steps`  
**Icon:** `list-ol`  
**Best For:** Step-by-step guides, workflows, getting started flows

**Key Features:**
- Numbered steps (1-5+)
- Icon support for each step
- Horizontal or vertical layout
- Connecting lines/arrows
- Optional CTA button at end

**Perfect For:**
- "How to Get Started" section
- Onboarding flow explanation
- Process walkthroughs

---

### 3. Stats Counter Widget
**Widget Name:** `stats-counter`  
**Icon:** `bar-chart`  
**Best For:** Key metrics, achievements, social proof

**Key Features:**
- Animated number counting on scroll-in
- 2, 3, or 4 column layouts
- Icon + Value + Label format
- Prefix/suffix support (R, +, %, min, etc.)
- Customizable colors

**Perfect For:**
- "By the Numbers" sections
- Achievement showcase (10 min setup, 99.9% uptime)
- Social proof (500+ customers)

**Key Features:**
- Simple list or categorized sections
- Checkmark icons (customizable)
- Optional expand/collapse for categories
- 1, 2, or 3 column layouts
- Clean, scannable design

**Perfect For:**
- "What's Included" sections
- Feature checklists
- "Everything You Need" lists
- Benefits organized by category

---

### 5. Feature Grid Widget
**Widget Name:** `feature-grid`  
**Icon:** `grid-3x3`  
**Best For:** Feature showcase, service highlights, capabilities overview

**Key Features:**
- 2, 3, or 4 column grid layouts
- Large circular icon backgrounds
- Title + description for each feature

## üé® Using the Widgets

### In the Page Builder
1. Open any landing page in edit mode
2. Click "Add Widget" in the page builder
3. Find your widget by name or icon:
   - `comparison-table` (table icon)
   - `process-steps` (list-ol icon)
   - `stats-counter` (bar-chart icon)
   - `benefits-checklist` (check-square icon)
   - `feature-grid` (grid-3x3 icon)
4. Drag and drop onto your page
5. Click to configure settings

### Configuration Options

**All widgets support:**
- Custom colors (title, subtitle, background, text)
- Padding adjustments
- Responsive behavior (automatic)

**Widget-Specific Settings:**

**Comparison Table:**
- Column count and highlighting
- Border visibility and radius
- Row hover effects
- Total/summary rows

**Process Steps:**
- Layout (horizontal/vertical)
- Connector lines (show/hide)
- Step numbering (automatic)
- Optional CTA button

**Stats Counter:**
- Column count (2-4)
- Animation on scroll (on/off)
- Prefix/suffix for values
- Icon colors and sizes

**Benefits Checklist:**
- Column count (1-3)
- Simple list or categorized
- Expand/collapse categories
- Icon customization

**Feature Grid:**
- Column count (2-4)
- Icon size (small/medium/large)
- Hover effects (on/off)
- Icon background colors

## üí° Tips for Best Results

**Comparison Table:**
- Keep to 2-3 columns max for readability
- Use highlighting on YOUR column
- Keep item labels short and scannable

**Process Steps:**
- 5 steps is ideal (max 7)
- Use relevant Bootstrap icons
- Keep descriptions under 100 characters
- Add CTA button at end for conversion

**Stats Counter:**
- 4 columns looks best on desktop
- Use round, impressive numbers
- Add prefixes/suffixes (R, +, %, etc.)
- Enable animation for impact

**Benefits Checklist:**
- Group related items into categories
- Use consistent icons (all checkmarks is fine)
- 2 columns balances space and readability
- Start with most important benefits

**Feature Grid:**
- 3 columns is standard (6-9 features total)
- Choose icons that clearly represent each feature
- Keep titles to 2-5 words
- Descriptions should be 1-2 sentences

## üöÄ Medium-Priority Widgets

### 6. Timeline Comparison Widget
**Widget Name:** `timeline-comparison`  
**Icon:** `clock-history`  
**Best For:** Comparing time-to-market, project timelines, speed comparisons

**Key Features:**
- Side-by-side timeline comparison (Traditional vs Platform)
- Breakdown of steps with individual durations
- Highlighted recommended option
- Total time summary per option
- Toggle step visibility

**Perfect For:**
- "2-4 months vs 10 minutes" comparison from landing content
- Implementation timeline comparisons
- Project phase breakdowns

**Configuration Tips:**
- Use dramatic time differences to highlight value
- Break down steps to show why traditional takes longer
- Highlight fastest option with green color

### 8. Video Embed Widget
**Widget Name:** `video-embed`  
**Icon:** `camera-video`  
**Best For:** Product demos, tutorials, testimonial videos

**Key Features:**
- YouTube and Vimeo support
- Responsive video player (16:9, 4:3, 1:1 aspect ratios)
- Autoplay, loop, mute options
- Custom video captions
- URL validation

**Perfect For:**
- Platform walkthrough videos
- Feature demonstrations
- Customer testimonial videos

**Configuration Tips:**
- Upload videos to YouTube/Vimeo first (better performance)
- 16:9 aspect ratio is standard
- Avoid autoplay with sound (most browsers block it)
- Keep videos under 3 minutes for best engagement

**Widget Name:** `logo-cloud`

**Best For:** Trust badges, partner logos, client logos, certifications

**Key Features:**

- Grayscale with color-on-hover effect

- Responsive grid

**Perfect For:**
- "Trusted By" sections

### 13. Pricing Cards Widget
**Widget Name:** `pricing-cards`  
**Icon:** `cash-coin`  
**Best For:** Pricing tiers, plan comparisons, subscription options

**Key Features:**
- 2-4 column layouts for different plans
- Popular plan highlighting with special styling
- Feature lists with included/excluded indicators
- Strike-through original prices for promotions
- CTA buttons per tier (different actions)
- Responsive card layouts
- Hover effects with elevation
- Optional billing toggle (monthly/annual)

**Perfect For:**
- Pricing pages and plan selection
- Feature comparison across tiers
- Subscription-based products
- Conversion optimization

**Configuration Tips:**
- Highlight the most popular/recommended plan
- Use 3 tiers (good-better-best psychology)
- Show savings with original prices (crossed out)
- Include clear feature lists (8-12 items per tier)
- Make CTAs action-oriented ("Start Free Trial" vs "Buy Now")
- Use checkmarks for included features, X for excluded
- Price anchoring: Show most expensive first to make others seem affordable

### 14. CTA Banner Widget
**Widget Name:** `cta-banner`  
**Icon:** `megaphone`  
**Best For:** Prominent calls-to-action, lead generation, conversions

**Key Features:**
- Full-width banner with maximum visibility
- Background options: solid color, gradient, or image
- Image overlay with adjustable opacity
- Light or dark text color modes
- Multiple CTA buttons (primary + secondary)
- Left/center/right text alignment
- 3 padding sizes (small/medium/large)
- Fade-in animations for content
- Responsive button layouts

**Perfect For:**
- Above-the-fold hero sections
- Mid-page conversion points
- Bottom-of-page final CTAs
- Free trial signups
- Demo requests
- Newsletter subscriptions

**Configuration Tips:**
- Use compelling, action-oriented headlines
- Limit to 2 buttons maximum (too many choices = no choice)
- Make primary action stand out visually
- Use high-contrast text colors for readability
- Gradient backgrounds more modern than solid colors
- For image backgrounds, use 0.5-0.7 overlay opacity
- Center alignment for maximum impact
- Large padding for hero sections, medium for mid-page

## üêõ Troubleshooting

**Widget not appearing?**
- Check widget-registry.ts imports
- Rebuild: `ng build`

**Styling looks off?**
- Check padding values (40-60px recommended)
- Verify color contrasts
- Test on mobile (all widgets are responsive)

**Animation not working?** (Stats Counter)
- Ensure `animateOnScroll` is `true`
- Check value format (numbers should be extractable)
